Perguntas Frequentes

Encontre respostas para as dúvidas mais comuns sobre nosso projeto. Clique em qualquer pergunta para ver a resposta detalhada.

Nosso projeto visa auxiliar a ONG Turma do Bem na gestão de voluntários e pacientes através de uma aplicação web responsável por manter e organizar os dados (Desde informações sobre os individuos, até as consultas e triagens), usar esses dados para auxiliar a operação da ONG, centralizar e organizar os meios de comunicação.

O projeto foi construído utilizando as seguintes tecnologias:

  • HTML5 - Para estruturação semântica do conteúdo
  • CSS3 - Estilização da plataforma web
  • Python - Lógica geral do backend
  • Flask - Framework para auxiliar o python
  • JavaScript - A ser implementado para dar responsividade à plataforma
  • MySQL - A ser implementado para organização dos dados

Sim! Optamos por já incluir responsividade para dispositivos Mobile, fazendo com que o projeto se adeque a partir de uma largura de 768px.

Utilizamos técnicas de CSS como:

  • Grid Layout e Flexbox para layouts fluidos
  • Media queries para adaptação em diferentes tamanhos de tela
  • Unidades flexivas como "rem, em, %.."

Sim, a acessibilidade é uma das prioridades em nosso projeto. Implementamos práticas para garantir que o conteúdo seja acessível para pessoas com deficiências.

Medidas de acessibilidade implementadas:

  • Estrutura HTML semântica com tags apropriadas
  • Atributos ARIA quando necessário
  • Contraste de cores adequado (WCAG AA)
  • Navegação por teclado funcional

Nosso projeto segue um roadmap estruturado em fases, com foco em criar uma solução completa para a gestão interna da ONG Turma do Bem. Estamos comprometidos com a entrega de uma plataforma robusta e eficiente.

Roadmap de Desenvolvimento:

  1. Fase 1 - Estrutura (Em andamento): Estabelecimento da arquitetura base, identidade visual e planejamento de integração entre módulos
  2. Fase 2 - Criação dos Módulos: Desenvolvimento dos módulos centrais incluindo cadastro de pacientes/dentistas/funcionários, módulo de relatórios, centro de comunicações e sistema de recomendação (Match) com parâmetros de prioridade, disponibilidade e região
  3. Fase 3 - Integrações e Testes: Conexão dos módulos ao banco de dados, ativação da comunicação entre módulos e testes completos de funcionalidade
  4. Fase 4 - Expansão e Manutenção: Implementação de novas funcionalidades baseadas no feedback da ONG e testes contínuos

Funcionalidades Técnicas Planejadas:

  • Modo Escuro: Tema alternativo para reduzir fadiga ocular e melhor experiência noturna
  • Otimização de Performance: Lazy loading, minificação de código e cache inteligente
  • Sistema de Match Inteligente: Algoritmo para conectar voluntários e pacientes de forma eficiente, com uma dashboard atraente
  • Repositórios Locais: Armazenamento de informações local para garantir integridade dos dados
  • Integração com APIs: Conexão com sistemas externos para ampliar funcionalidades
  • Backup Automatizado: Sistema de segurança para proteção dos dados com redundância local e em nuvem
